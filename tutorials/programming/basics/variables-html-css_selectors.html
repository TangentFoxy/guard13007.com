Programming Basics: Variables, HTML, CSS Selectors

So I'm going to try to get straight to the core concepts of programming, and skip over getting buried by syntax rules. Every language has its own rules, but the concepts are universal. Here are a couple definitions:

variable
A named value. Think of a space, with some name on it. That's a variable. What you put inside it is the value. Think math "x = 5" or whatever, except x can equal a string, or an image, or anything.
keyword
Some words have special meaning to the computer, and you can't use them as a name for a variable. These are called keywords. An example common keyword is "function" which is often used to state what a function is. (Don't worry about "what" a function is yet.)
object
Essentially, a named collection of variables. (You will find different definitions for this, I'm using a very generic definition.)
(also: "a block of stuff" in laymen's terms)

As long as you understand a "variable" is a thing that can hold some value, and a keyword is a word with special meaning to the computer, you're good to go. Here's some code:


            info = {
                domain = {-100, 100},
                detail = 1
            }

So what does this do? Let's take it line by line:

info = { "info" is a variable name, the "=" means we're putting a value in it, and the "{" (bracket) means that it's going to be an object.

domain = {-100, 100}, "domain" is another variable. We're putting a variable in an object. This variable is itself an object. The commas are used to tell the computer when we're done with one variable (or object) and are defining another. In this case, we're putting some numbers in here.

detail = 1 Here we define one more variable with a one. As this is the last variable, we don't have a comma following it, but that's not important right now.

} And we're done. In this language, all brackets must pair up. (A lot of languages are like that!)

If you're worrying about the rules or how that works at this point, stop. It would waste far too much time to try to go through all that right now. The main lesson here is that we have a block of stuff called "info", and within that block of stuff is a thing called "domain" with two numbers in it, and another thing called "detail" with the value of one.

Now, HTML

Let's start again with an example:


            <html>
                <head>
                    <title>Page Title!</title>
                </head>
                <body>
                    <h1>A Header</h1>
                    <p>A paragraph of text.</p>
                </body>
            </html>

First, what's the same?

  • We have "objects" (though we now call them "elements" or "tags"). <html> all the way to </html> is one big block of stuff - everything else is in there!
  • We have values, like A Header. Some of these would show on the webpage if you opened it in a browser, like this.

Second, what's different?

  • We don't have any variables. We have "tags" defined by their keywords. Anything in angled brackets (the < and > symbols) is a keyword with specific meaning. (For a huge list of them with a lot of examples, click here.)
  • This is a complete, useful document. The other example was "here's some data" without doing anything. HTML is about displaying data. You say what the data is by which tags you use, and the browser displays it according to rules about what each tag means.

At this point, you're probably wondering what can I do? and what can I not do?

What you can do is use any tag listed here in any order, with anything within the <open> and </close> ends of the tag... except you can't. There are some specific rules with certain tags, and some tags don't have a closing tag.

CSS Selectors

CSS is used to style websites. If you clicked on the example page above, you noticed it was very bland. CSS is a langauge for modifying how HTML is displayed.

(Real quick, a great introduction to CSS to read is available here. You could even read their full tutorial if you want, and you'll know more than the brief part I'm going over here.)


            * {
                color: red;
            }
            a.no_underline {
                text-decoration: none;
            }
            a .no_underline {
                text-decoration: none;
            }

Okay, there's a lot in here, but we're focusing on the selectors, which is the part before the brackets on each of these blocks. The first one (*) is pretty simple, it selects everything. In this case, we're telling everything to be red.

The second one is a little more complicated. The first part (the "a") is selecting "a" tags (anchor tags). The period means that the text immediately following the period is a class name. In this case, we have a class named "no_underline".

class
A class simply refers to a collection of tags that are similar in some way in what CSS wants to do with them.
(There are some rules to how class names can be chosen, like what's mentioned here, just don't get bogged down in the details yet! (By the way, that's a great site for answers, but only as long as you carefully read responses, there's just as many wrong answers or bad answers as good ones!))

The next definition is the exact same thing, I'm just showing you that you can have a space there..but not after the period!


            p {
                font-size: 30px;
            }
            .class {
                color: blue;
            }
            .class {
                font-weight: bold;
            }

The "p" is just selecting another tag, this time the paragraph tag. Next, to emphasize that you can re-use these selectors all over the place, we have two definitions of a class called "class". This is the same thing as:


            .class {
                color: blue;
                font-weight: bold;
            }

But sometimes, you need to combine a whole bunch of these!


            .blacklist > ol > li {
                background: black;
                /* this only selects ordered list items inside a "blacklist" class */
            }

Here's an example using a class and two tags, as well as > which modifies the meaning of other selectors to say that we're looking for whatever is on the right, only if it is in what's on the left.
